Por que precisamos ir além das proposições?

A Lógica Proposicional é poderosa, mas tem uma limitação fundamental: ela trata cada afirmação como um bloco atômico, sem conseguir "entrar dentro" das frases. Observe o problema:

Premissa 1: "Sócrates é humano."
Premissa 2: "Todo humano é mortal."
Conclusão: "Sócrates é mortal."

→ A Lógica Proposicional NÃO consegue provar essa conclusão!

Para isso, precisamos de uma lógica que entenda objetos, propriedades e relações. Essa é a Lógica de Predicados — também chamada de Lógica de Primeira Ordem (LPO) ou Cálculo de Predicados.

O grande salto conceitual

Lógica Proposicional
Trabalha com proposições inteiras: p, q, r...
Não enxerga o interior das frases.
Não fala de "todos" ou "existe".
Universo fechado e fixo.
Lógica de Predicados
Decompõe frases em objetos e propriedades.
Usa variáveis para representar objetos.
Tem quantificadores: ∀ e ∃.
Pode raciocinar sobre domínios infinitos.
📌 A Lógica de Predicados é a base da matemática formal, da programação em lógica (Prolog), de bancos de dados relacionais, e de sistemas de IA como provadores de teoremas.

Componentes básicos

1
Termos: constantes (objetos específicos: Sócrates, 5, Brasil), variáveis (x, y, z) e funções (pai(x), soma(x,y)).
2
Predicados: propriedades ou relações aplicadas a termos. Humano(x), Maior(x,y), Par(n).
3
Quantificadores: ∀ (para todo) e ∃ (existe). Permitem fazer afirmações sobre coleções de objetos.
4
Conectivos lógicos: os mesmos da LP — ¬, ∧, ∨, →, ↔ — mas agora aplicados a fórmulas com predicados.
5
Domínio de interpretação: o conjunto de objetos sobre os quais as variáveis podem assumir valores. Ex: ℕ, ℝ, conjunto de pessoas.

O que é um Predicado?

Um predicado é uma função que recebe um ou mais termos e retorna um valor-verdade (V ou F). É como uma frase com "lacunas" para serem preenchidas por objetos.

P(x) → "x é par" — predicado unário
Maior(x,y) → "x é maior que y" — predicado binário
Entre(x,y,z) → "x está entre y e z" — predicado ternário

Quando substituímos as variáveis por constantes, o predicado se torna uma proposição com valor-verdade definido.

Aridade dos Predicados

AridadeTipoExemploResultado

Constantes, Variáveis e Funções

C
Constantes nomeiam objetos específicos do domínio: socrates, 0, brasil. Sempre com letra minúscula por convenção.
V
Variáveis representam objetos arbitrários do domínio: x, y, z. Seu valor é determinado pelos quantificadores ou por substituição.
F
Funções mapeiam objetos em objetos: pai(x), soma(x,y), quadrado(n). Não retornam V/F — retornam objetos do domínio.
⚠️ Atenção crucial: Predicados retornam V ou F. Funções retornam objetos. Não confunda!

Igualdade

A igualdade x = y é um predicado binário especial, presente em quase todas as teorias. Permite expressar unicidade e identidade de objetos:

"Existe um único x tal que P(x)":
∃x (P(x) ∧ ∀y (P(y) → y = x))

Os Dois Quantificadores

Os quantificadores são o coração da Lógica de Predicados — é o que a torna incomparavelmente mais expressiva que a Lógica Proposicional. Clique em cada um.

∀
Quantificador Universal
"para todo" / "para qualquer"
∃
Quantificador Existencial
"existe" / "existe pelo menos um"

∀ — Quantificador Universal

A fórmula ∀x P(x) afirma que para todo objeto x do domínio, P(x) é verdadeiro. É equivalente a uma grande conjunção sobre todos os elementos do domínio.

Domínio D = {a, b, c}:
∀x P(x) ≡ P(a) ∧ P(b) ∧ P(c)
Para falsificar ∀x P(x), basta encontrar um único elemento d do domínio tal que P(d) seja falso. Esse elemento é chamado de contraexemplo.

Exemplos:

∀x (Humano(x) → Mortal(x)) — "Todo humano é mortal"

∀x (Par(x) → ∃y (x = 2·y)) — "Todo par é o dobro de algum inteiro"

Uso com condicional: o ∀ é tipicamente combinado com →. Escrever ∀x (Humano(x) ∧ Mortal(x)) diria "todo objeto do domínio é simultaneamente humano e mortal" — muito mais forte e geralmente errado!

∃ — Quantificador Existencial

A fórmula ∃x P(x) afirma que existe pelo menos um objeto x no domínio tal que P(x) é verdadeiro. Equivale a uma grande disjunção.

Domínio D = {a, b, c}:
∃x P(x) ≡ P(a) ∨ P(b) ∨ P(c)
Para verificar ∃x P(x), basta encontrar um único elemento d tal que P(d) seja verdadeiro. Esse elemento é chamado de testemunha.

Exemplos:

∃x Primo(x) — "Existe (pelo menos) um número primo"

∃x (Estudante(x) ∧ Passou(x)) — "Algum estudante passou"

Uso com conjunção: o ∃ é tipicamente combinado com ∧. Escrever ∃x (Estudante(x) → Passou(x)) seria satisfeito por qualquer não-estudante, pois F→F=V. Isso raramente é o que se quer dizer!

Quantificadores Aninhados

É possível (e muito comum) aninhar quantificadores. A ordem importa — trocar a ordem pode mudar completamente o significado.

FórmulaLeituraDomínio ℕ
∀x ∃y Maior(y,x)Para todo x existe y maior que xV
∃y ∀x Maior(y,x)Existe y que é maior que todos os xF
∀x ∀y Amigo(x,y)Todos são amigos de todosF*
∃x ∃y Amigo(x,y)Existem dois que são amigosV*

* Resultado depende da interpretação de Amigo(x,y)

⚠️ Erro clássico: ∀x ∃y ≠ ∃y ∀x. No primeiro, y pode depender de x. No segundo, um único y serve para todos os x — condição muito mais forte!

Variáveis Livres e Ligadas

Uma variável é ligada quando está no escopo de um quantificador. É livre caso contrário.

∀x (P(x) → Q(x, y))
↑ ligada ↑ ligada ↑ livre

Uma fórmula com variáveis livres é chamada de fórmula aberta — não tem valor-verdade definido sem saber o valor de y. Uma fórmula sem variáveis livres é uma sentença e tem valor-verdade definido.

Construindo Fórmulas Bem Formadas (FBFs)

Uma Fórmula Bem Formada (FBF) em Lógica de Predicados é construída pelas seguintes regras:

1
Se P é um predicado n-ário e t₁,...,tₙ são termos, então P(t₁,...,tₙ) é uma FBF (fórmula atômica).
2
Se A é uma FBF, então ¬A é uma FBF.
3
Se A e B são FBFs, então A ∧ B, A ∨ B, A → B, A ↔ B são FBFs.
4
Se A é uma FBF e x é uma variável, então ∀x A e ∃x A são FBFs.

Traduzindo Linguagem Natural → Lógica

Esta é a habilidade mais importante e mais praticada em provas. Veja os padrões:

Linguagem NaturalFórmula Lógica
Todo A é B∀x (A(x) → B(x))
Algum A é B∃x (A(x) ∧ B(x))
Nenhum A é B¬∃x (A(x) ∧ B(x))
Nem todo A é B¬∀x (A(x) → B(x))
Só A's são B's∀x (B(x) → A(x))
Existe exatamente um A∃x (A(x) ∧ ∀y(A(y)→y=x))

Exemplo Clássico — Sócrates

Humano(s) — "Sócrates é humano" [s = sócrates]
∀x (Humano(x) → Mortal(x)) — "Todo humano é mortal"
─────────────────────────────
Mortal(s) — "Sócrates é mortal" ✓ (por Modus Ponens)
Agora conseguimos provar formalmente o que a Lógica Proposicional não conseguia!

Interpretação e Satisfatibilidade

Uma interpretação (ou estrutura) consiste em: (1) um domínio D não vazio, (2) uma atribuição de objetos de D às constantes, (3) uma atribuição de relações sobre D aos predicados.

Uma fórmula é satisfatível se existe alguma interpretação que a torna verdadeira. É válida (tautologia) se é verdadeira em toda interpretação possível.

⚠️ Na LPO, verificar validade é semidecidível — podemos provar que algo é válido, mas não podemos sempre provar que é inválido. Isso é o Teorema da Incompletude de Gödel!

Negação de Quantificadores — Leis de De Morgan Generalizadas

Assim como a Lógica Proposicional tem as Leis de De Morgan para ∧ e ∨, a Lógica de Predicados tem equivalências análogas para os quantificadores. São essenciais para manipular e simplificar fórmulas.

¬∀x P(x)
≡ ∃x ¬P(x)
"Não é o caso que todo x satisfaz P" = "existe x que não satisfaz P"
¬∃x P(x)
≡ ∀x ¬P(x)
"Não existe x que satisfaz P" = "para todo x, P é falso"
💡 Mnemônico: ao empurrar a negação para dentro, o quantificador "vira" — ∀ vira ∃ e ∃ vira ∀.

Negação de Fórmulas Complexas — passo a passo

Negar: "Todo humano é mortal" → ∀x (H(x) → M(x))

¬∀x (H(x) → M(x))
≡ ∃x ¬(H(x) → M(x)) [¬∀ vira ∃]
≡ ∃x ¬(¬H(x) ∨ M(x)) [eliminação do →]
≡ ∃x (H(x) ∧ ¬M(x)) [De Morgan]

Leitura: "Existe algum humano que não é mortal"

Negar: "Existe um estudante que passou" → ∃x (E(x) ∧ P(x))

¬∃x (E(x) ∧ P(x))
≡ ∀x ¬(E(x) ∧ P(x)) [¬∃ vira ∀]
≡ ∀x (¬E(x) ∨ ¬P(x)) [De Morgan]
≡ ∀x (E(x) → ¬P(x)) [eliminação do ∨]

Leitura: "Nenhum estudante passou"

Negar: "Para todo x existe y maior" → ∀x ∃y Maior(y,x)

¬∀x ∃y Maior(y,x)
≡ ∃x ¬∃y Maior(y,x) [¬∀ vira ∃]
≡ ∃x ∀y ¬Maior(y,x) [¬∃ vira ∀]

Leitura: "Existe x tal que nenhum y é maior que x"
(i.e., existe um máximo no domínio)

Hierarquia da Lógica

1
Lógica Proposicional (LP): sem variáveis, sem quantificadores. Base.
2
Lógica de Primeira Ordem (LPO): variáveis e quantificadores sobre objetos. O que estudamos aqui.
3
Lógica de Segunda Ordem: quantificadores também sobre predicados e funções. Mais expressiva, menos tratável.
4
Lógicas de Ordem Superior: quantificam sobre tudo. Muito expressivas mas computacionalmente intratáveis.

Prolog — Programação em Lógica

Prolog é uma linguagem de programação baseada diretamente em Lógica de Predicados. Fatos e regras são literalmente predicados:

humano(socrates). /* Humano(socrates) */
mortal(X) :- humano(X). /* ∀x Humano(x) → Mortal(x) */
?- mortal(socrates). /* Consulta: Mortal(socrates)? */
true. /* Resposta: V */

Banco de Dados Relacionais

A Álgebra Relacional e o SQL têm fundamentos na Lógica de Predicados. Uma tabela é uma extensão de um predicado; uma query é uma fórmula:

Tabela Estudante(x) → predicado unário
Tabela Nota(x, disciplina, nota) → predicado ternário

SELECT * FROM Estudante WHERE aprovado = true
≡ { x | Estudante(x) ∧ Aprovado(x) }
≡ ∃x (Estudante(x) ∧ Aprovado(x))

Inteligência Artificial — Representação do Conhecimento

Sistemas especialistas e ontologias usam Lógica de Predicados para representar conhecimento do mundo:

∀x (Ave(x) ∧ ¬Pinguim(x) → Voa(x)) /* Aves voam, exceto pinguins */
∀x (Pinguim(x) → Ave(x)) /* Pinguins são aves */
Ave(tweety) ∧ ¬Pinguim(tweety)
─────────────────────────────────
Voa(tweety) ✓

Verificação Formal de Software

Ferramentas como Coq, Isabelle e Z3 usam LPO para provar matematicamente que programas estão corretos:

Pré-condição: ∀i (0 ≤ i < n → A[i] ≥ 0)
Pós-condição: soma = ∑ᵢ A[i]

→ Prova formal que o algoritmo é correto para qualquer entrada válida
💻 Na computação moderna: SAT solvers (que verificam satisfatibilidade) e SMT solvers (que estendem SAT com aritmética) são aplicações diretas da LPO usadas em compiladores, segurança e hardware.
